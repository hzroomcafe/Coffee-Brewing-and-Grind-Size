連接後端：讓投稿送到你的 Google
並寄信通知你

這頁帶你做完設定。你不需要懂程式——照著做、用複製鈕，最後把一個網址傳給我就完成。全程約 10 分鐘。

使用者填表 → 你的免費後端 → 寫進試算表 ＋ 寄信給你

因為網頁放在 GitHub（只能顯示、不能收資料），我們借 Google 自己的免費服務「Apps Script」當中間人。設定一次，以後都自動運作。

1

加一個收投稿的分頁

1 分鐘

打開你的 Google 試算表，左下角按 ＋ 新增一個分頁，命名為 投稿。
在第一列（最上面那排格子）由左到右貼上這 11 個標題，一格一個：

第一列標題（用 Tab 分隔，可整段貼上自動分欄） 
時間	店名	區域	地址	緯度	經度	推薦人	推薦原因	標籤	連結	狀態
提示：複製後在「投稿」分頁點 A1 格貼上，Google 會自動把 11 個標題分到 A～K 欄。
2

打開 Apps Script，貼上程式碼

2 分鐘

在試算表最上方選單點：

擴充功能 › Apps Script

會開啟一個新分頁、裡面有幾行預設程式碼。把它全部選起來刪掉，然後按下面的鈕複製、貼上整段：

程式碼.gs 
// ===== 只改這三行（下一步說明）=====
const SHEET_ID   = '你的試算表ID';
const SHEET_NAME = '投稿';
const NOTIFY_TO  = '你的email@gmail.com';

const MAX_PER_FP_HOUR = 5;

function doPost(e) {
  try {
    const d = JSON.parse(e.postData.contents);
    if (d.website) return ok();
    if (!d.name || !d.by || !d.reason || !d.tags) return ok();
    const reason = String(d.reason);
    if (reason.length < 5 || reason.length > 300) return ok();

    const props = PropertiesService.getScriptProperties();
    const key = 'rl_' + (d._fp || 'anon');
    const now = Date.now();
    const hist = JSON.parse(props.getProperty(key) || '[]')
                 .filter(t => now - t < 3600000);
    if (hist.length >= MAX_PER_FP_HOUR) return ok();
    hist.push(now); props.setProperty(key, JSON.stringify(hist));

    const sh = SpreadsheetApp.openById(SHEET_ID).getSheetByName(SHEET_NAME);
    const recent = sh.getDataRange().getValues().slice(-60);
    const dup = recent.some(r => r[1] === d.name && r[6] === d.by &&
                (now - new Date(r[0]).getTime()) < 86400000);
    if (dup) return ok();

    sh.appendRow([new Date(), d.name, d.area||'', d.address||'',
      d.lat||'', d.lng||'', d.by, reason, d.tags, d.link||'', '待審核']);

    MailApp.sendEmail(NOTIFY_TO, '☕ 新咖啡廳推薦待審核',
      '店名：'+d.name+'\n推薦人：'+d.by+'\n原因：'+reason+
      '\n標籤：'+d.tags+'\n地址：'+(d.address||'(無)')+
      '\n座標：'+d.lat+','+d.lng+
      '\n\n到試算表把「狀態」改成「已通過」即可上架。');
    return ok();
  } catch (err) { return ok(); }
}
function ok(){ return ContentService.createTextOutput('{"ok":true}')
  .setMimeType(ContentService.MimeType.JSON); }
3

填入你自己的資料（橘色那兩行）

2 分鐘

程式碼最上面有橘底標示的地方，改成你的：

SHEET_ID打開試算表時，網址 /d/ 和 /edit 中間那串長字母，就是 ID。複製貼進去（保留兩邊的單引號）。
NOTIFY_TO你想收「有新推薦」通知信的 Gmail。
SHEET_NAME 不用改，維持 '投稿' 就好（要和步驟 1 的分頁名一樣）。
4

部署成「網頁應用程式」

2 分鐘

右上角點 部署 → 新增部署作業。齒輪選「類型 → 網頁應用程式」，下面兩個設定照這樣選：

這樣選執行身分：我（你自己的帳號）
這樣選誰可以存取：任何人

按「部署」。「任何人」是必要的——這樣你的網頁才能把投稿送進來；但別擔心，這支程式只能「新增一筆待審核投稿」，讀不到你其他資料。

5

授權（第一次會問）

1 分鐘

會跳出授權視窗：選你的 Google 帳號 → 看到「Google 尚未驗證這個應用程式」時，點 進階 → 前往「專案」（不安全） → 允許。

為什麼說「不安全」？因為這是你自己寫的、沒上架審查的小程式，Google 一律這樣提醒。它只會寫你自己的試算表＋用你的信箱寄通知給你，安全無虞。
6

複製網址 — 這就是要給我的東西

10 秒

部署成功後，會跳出一個視窗，裡面有「網頁應用程式網址」和一個「複製」按鈕。按那個複製鈕。網址長這樣：

把這個傳到對話框給我

就傳這一行網址，其他都不用給

它以 https://script.google.com/macros/s/ 開頭、以 /exec 結尾：

https://script.google.com/macros/s/AKfycbx9...一長串.../exec

把這串網址貼到我們的對話框傳給我。我收到後會幫你接進推薦表單、加上裝置防護、再測一次送出，整套就完成了。
